Envoyez vos fichiers directement

Vos fichiers passent d'un appareil à l'autre sans jamais toucher un serveur. Personne ne peut les intercepter, les lire, ou les stocker. Pas de limite de taille, pas de compte requis.

Le transfert est protégé par deux couches de sécurité indépendantes. Les clés sont générées sur votre appareil et détruites à la fin de chaque session.

Ce qui protège vos fichiers
  • Connexion directe entre les deux appareils, sans serveur intermédiaire
  • Double couche de protection indépendante sur toutes les données
  • Clés uniques générées localement, détruites après utilisation
  • Empreinte de vérification pour confirmer l'authenticité de la connexion
  • Code source ouvert et vérifiable sur GitHub

À propos de votre adresse IP

Les connexions directes révèlent votre adresse IP à votre correspondant. Si vous avez besoin d'un anonymat complet, utilisez Transfer via Tor Browser ou un VPN qui protège contre les fuites WebRTC.

Protocoles utilisés

Les mêmes standards que les messageries les plus sécurisées

X25519

Échange de clés sur courbe elliptique moderne, plus résistant que les standards gouvernementaux.

AES-256-GCM

Protection des données avec authentification intégrée et vérification d'intégrité.

HKDF-SHA-256

Dérivation de clés uniques pour chaque opération à partir du secret partagé.

DTLS 1.2+

Seconde couche de protection sur le transport WebRTC.

Perfect Forward Secrecy

Clés éphémères détruites après chaque session.

IV aléatoire par bloc

Vecteur d'initialisation unique pour chaque morceau de fichier.
